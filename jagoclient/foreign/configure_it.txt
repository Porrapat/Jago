Configurare le connessioniPer configurare la tua connessione al server premi "Modifica" o"Aggiungi" nella finestra "Connessioni Server". Apparira' una finestradove devi inserire i seguenti dati :- "Nome Connessione": questo e' il nome che apparira' nella lista  delle "Connessioni server".- "Server": il nome del server (sotto troverai una lista di server  utilizzabili)- "Porta": la porta utilizzata dal server (vedi sotto). Se pensi di  connetterti utilizzando una sessione telnet inserisci 23 (per  maggiori informazioni leggi gli aiuti relativi alle Firewalls).- "Utente" (user name): il nome che utilizzi sul server. Se ti  connetti come ospite (guest) non inserire nulla.- "Password": la password che utilizzi per il login al server. Se  lasci questo campo vuoto la password ti verra' richiesta ogni volta  che ti connetterai.- "Stile Mossa": Non modificare il valore di default ("mossa") a meno  che tu non sappia esattamente quello che stai facendo.- "Codifica": questo campo determina il modo in cui le stringhe  vengono tradotte tra Java e il server. Quando aggiungi una nuova  connessione viene inserito il codepage di default. Per l'Europa il  codepage e' 8859_1 o Cp1252. Se lasci vuoto questo campo verra'  utilizzato il codepage di default. E' comunque possibile utilizzare  il codice ASCII. Se al codepage preponi !, non verra' utilizzato il  filtro Telnet.Di seguito trovi gli indirizzi e le porte dei server attualmente esistenti:igs.joyjoy.net 6969nngs.cosmic.org 9696lgs.hinet.net 9696wing.brlnet.net 1515JagoClient e' preconfigurato per questi server.Problemi con le codificheAlcune pagine di codice contengono piu' di 256 caratteri.  Per evitareerrori usa la codifica corretta o una codifica non critica come 8859_1o ASCII. Come ulteriore difficolta' il server puo' inviare caratteriTelnet i quali possono confondere il decodificatore.  Questi sononormalmente filtrati, il che puo' ancora confondere ildecodificatore. Se hai dei problemi con una pagina di codice, prova apreporre il carattere ! alla pagina di codice (ad esempio"!Cp1552"). Ora, Jago non cerchera' di filtrare i comandi Telnet. Seanche questo non funziona, usa "!ASCII", e poi "ASCII".
