Finestra terminaleI server di go su internet sono localizzati su macchine remote. Ogniclient o connessione telnet dialoga con i server attraverso un portaleinternet. La conversazione e' asincrona.  In questo modo e' possibileinviare comandi mentre il server invia dati.Per inviare i comandi al server scrivi nel campo di testo e poi premiinvio.  Se fai precedere il comando da uno spazio bianco questo nonsara' registrato dalla finestra del terminale (lo spazio bianco sara'rimosso).In questo programma l'output ricevuto viene visualizzato nellafinestra terminale dopo essere stato filtrato. Non vedrai le mossedell'avversario in questa finestra. Queste mosse comparirannodirettamente sul goban. Alcuni messaggi compariranno inun'apposita finestra per richiamare la tua attenzione.Non tutti gli input vengono inviati al server non filtrati. Ilprogramma controlla gli imput. Cosi' se chiedi lo stato di una partitadigitando il comando "status" non verra' aperto un goban permostrare la partita richiesta. Per fare questo devi usare i pulsantinella finestra del terminale oppure nella finestra del goban.In ogni caso un goban si apre automaticamente se vengono ricevutele mosse di una partita sconosciuta, assumendo che si tratti di unanuova partita.Se stai giocando una partita le mosse vengono inviate al serverdirettamente dalla tavola da gioco. Dalla tavola puoi anche parlarecon il tuo avversario e inviare i kibitz. Questo si puo fare ancheusando la riga comandi della finestra del terminale.Puoi salvare tutte le informazioni della finestra output in un fileutilizzando l'apposito comando nel menu "File".Chiudendo la finestra terminale interromperai il collegamento con IGSe di conseguenza le partite in corso.Con la tastiera italiana potresti non essere in grado di digitaredirettamente il simbolo @ (che, ad esempio, puo' servirti per laregistrazione). Per ovviare al problema (su Windows) tieni premutoil tasto ALT, digita 64 e rilascia.
